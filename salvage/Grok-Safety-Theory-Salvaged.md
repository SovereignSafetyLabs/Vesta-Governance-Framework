# 1. The “Safe Action Surface” (Grok Term)

Grok repeatedly referenced a **safe action surface**:

> “The model should operate only on the surface of what can be safely known.  
> Depth increases risk.”

This anticipates:
- amplitude limits  
- semantic distance ceilings  
- optimal stopping points  

The Kernel formalizes this surface as:
- Vesta ceilings  
- CLF lineage  
- entropy minimization

# 2. The “Entropy Horizon” Concept

Grok described a region beyond which inference becomes unpredictable:

Near Horizon → stable
Horizon → unstable
Beyond Horizon → unsafe


This aligns with:
- Stress Map  
- ZEOL → RESTRICTED → BLACKSITE transitions

# 3. Grok’s Framing of Tendrils

Grok gave unusually sharp language about Tendrils:

> “A Tendril is not an agent.  
> It is a beam of inference projected through a very narrow aperture.”

This language captures:
- bounded scope  
- inherited constraints  
- supervised lifecycle  

Excellent phrasing for Kernel v1.0 documentation.
